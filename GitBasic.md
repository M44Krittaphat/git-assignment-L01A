# Git คืออะไร?
Git เป็นเครื่องมือที่มาช่วยจัดการเวอร์ชั่นของโค้ดให้กับเรา โดยจะมีการเก็บประวัติว่าไฟล์แต่ละไฟล์ถูกสร้าง/ลบ/แก้ไขโดยระบุด้วยว่าใครทำ ทำเมื่อไหร่ และทำอะไรเอาไว้ทั้งหมด ทำให้เราสามารถติดตามการเปลี่ยนแปลงของโค้ดได้ตลอด หรือแม้กระทั่งย้อนเวลาโค้ดกลับไปก่อนตอนที่จะพังก็ยังทำได้
# คำสั่งพื้นฐา Git มีอะไรบ้าง?
## Git Init
เป็นคำสั่งที่มีไว้ใช้งานที่โฟลเดอร์งานนั้นเช่น
```
cd workspace
git init
```
## Git Add
ใช้สำหรับเพิ่มไฟล์เข้าไปใน stage เช่น
```
git add main_file.html
```
หรือถ้าต้องการเพิ่มทั้งโฟลเดอร์ก็สามารถใช้คำสั่งต่อไปนี้ได้
```
git add .
```
## Git Status
ใช้สำหรับตรวจสอบว่ามีไฟล์ไหนที่ยังไม่ได้ add หรือมีการเปลี่ยนแปลงแล้วยังไม่ถูกจัดการบ้าง
```
git status
```
## Git Reset
ใช้สำหรับการย้อน version ไฟล์ให้กลับไปอยู่ในสภาพเดิมก่อนที่จะถูก add ไฟล์เข้าสู่ stage
```
git reset
```
## Git Commit
ใช้สำหรับการยืนยันการเปลี่ยนแปลงไฟล์ที่ถูก add ลงใน stage โดยสามารถใส่ comment โดยการเติม -m ด้านหลัง เช่น
```
git commit -m “Add Hello World to main”
```
## Git Clone
ใช้สำหรับการก้อปปี้โปรเจคจำลองมาจากโปรเจคหลัก โดยเราจะสามารถเปลี่ยนแปลงไฟล์ในโปรเจคจำลองนี้ได้โดยที่ไม่กระทบโปรเจคหลัก
```
git clone
```
## Git Push
ใช้สำหรับการส่งไฟล์ที่ commit แล้วเข้าสู่ remote repository
```
git push
```
## Git Fetch
ใช้สำหรับตรวจสอบว่าไฟล์ ใน local กับ remote มีความแตกต่างกันหรือไม่ ไฟล์ไหนใหม่กว่า หรือล้าสมัยอยู่กี่ commit
```
git fetch
```
## Git Pull
ใช้สำหรับการดึงไฟล์ที่มีการเปลี่ยนแปลงใน remote มาเปลี่ยนแปลงใน local ซึ่งเป็นการใช้คำสั่ง git fetch ตามด้วย git merge
```
git pull
```
